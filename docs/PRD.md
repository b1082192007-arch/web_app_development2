# 產品需求文件 (PRD) - 任務管理系統

## 1. 專案概述
本專案為一個任務管理系統（Todo App），包含後端 Flask 伺服器與 SQLite 資料庫，以及前端 HTML/CSS/JS 的現代化介面。旨在協助個人使用者管理日常工作與待辦事項。

## 2. 目標受眾 (Target Audience)
- 需要管理日常工作、追蹤個人目標或專案進度的個人用戶。

## 3. 核心功能需求
1. **新增任務**：使用者可以輸入文字來新增待辦任務。
2. **標記任務完成**：使用者可以將任務切換為「已完成」或「未完成」狀態。
3. **刪除任務**：使用者可以刪除不再需要或輸入錯誤的任務。
4. **顯示所有任務清單**：從後端資料庫取得所有任務並呈現在畫面上。
5. **依完成狀態篩選**：前端可依據「全部 (All)」、「進行中 (Active)」、「已完成 (Completed)」來篩選顯示的任務。

## 4. 非功能性需求
- **使用者體驗 (UX) / 介面設計 (UI)**：需提供流暢的微動畫、現代化設計。
- **資料持久化**：使用 SQLite 資料庫來儲存任務資料，並透過 Flask 建立 API 進行串接，確保資料穩定且不會因為前端刷新而遺失。
- **響應式設計**：支援桌面、平板與手機等不同螢幕尺寸。
